前端包管理器的说明

发表于 2020-03-25 | 更新于: 2020-03-29 | 分类于 包管理器

安装模块的方式

  • 全局安装：添加到电脑环境变量中，可直接使用执行指令，可在不同项目间使用
  • 本地安装：添加到对应项目的依赖中，仅在当前项目使用

 最近在看 webpack 的书籍，里面提到安装 webpack 时推荐本地安装，原因如下两点：

  • 全局安装的话在与他人协作开发时，可能会因个人电脑 webpack 版本有差异，导致输出结果不同
  • 部分依赖于 webpack 的插件会调用项目中的 webpack 的内部模块，这种情况需要在项目本地安装 webpack，如果本地与全局都有，则容易造成混淆

npm 与 npx 的区别

npm 的 m 是 Management，npx 的 x 可以理解为 eXecute。

  1. npm(node package manager)：因社区文化所衍生的包管理器，有以下特点：
    1.1 有了 npm，不再需要像以前一样跑各个官网去下载需要的代码。比如使用 jq，要去 jq 官网下载然后引入。
    1.2 可以通过 package.json 更高效的管理依赖包，通过安装的方式，npm 就会帮忙下载对应代码，下载好的代码会出现在 node_modules 文件夹中，然后就可以引入使用了。

  2. npx：是一个工具，npm v5.2.0 引入的一条命令（npx），一个 npm 包执行器，指在提高从 npm 注册表使用软件包时的体验。有以下特点：
    2.1 临时安装可执行依赖包，不用全局安装，不用担心长期的污染。
    2.2 可以执行依赖包中的命令，安装完成自动运行。
    2.3 自动加载 node_modules 中依赖包，不用指定$PATH。
    2.4 可以指定 node 版本、命令的版本，解决了不同项目使用不同版本的命令的问题。
